## 2024-05-24 - TypedArray vs Standard Array and Higher-Order Array Methods
**Learning:** In code executing at high frequency (like the visual `scoreFrame` running every 2s on video frames), standard dynamic JS arrays (`[]`) coupled with `.push()`, `.reduce()`, and `.filter()` incur massive Garbage Collection (GC) pauses and high overhead. `Float32Array` with simple loops consistently outperformed standard approaches by ~10-15x in Node.js profiling. Specifically, `Float32Array.sort()` is inherently numeric and optimized, avoiding the need for a comparator function `(a,b) => a-b`.
**Action:** When working on pixel/image data or math-heavy arrays of known sizes, always pre-allocate `Float32Array` (or `Uint8Array`) and use standard `for` loops instead of mapping/reducing over standard dynamic arrays. Calculate sums (like Red/Green/Blue channels) during the single initial pass rather than storing the values for subsequent reductions.
