## 2024-05-19 - Client-side image processing performance
**Learning:** In `src/app/page.tsx`, the `scoreFrame` function is called frequently to evaluate camera frame quality. The original implementation used dynamic JavaScript arrays (`number[]`) to store pixel data (e.g., `grayscale`, `gradientMagnitudes`) and heavily relied on array methods like `.reduce()` and `.filter()`. Because this runs in a hot loop (potentially every frame), it caused massive garbage collection (GC) overhead and CPU spikes.
**Action:** For client-side image processing or any high-frequency numeric array operations, always use Typed Arrays (e.g., `Float32Array`, `Int32Array`) pre-allocated to the exact needed size. Replace expensive array methods (`reduce`, `filter`) with simple `for` loops and running sums.
