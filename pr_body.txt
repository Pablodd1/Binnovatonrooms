💡 **What:**
Updated FastAPI async endpoints (`/detect` and `/detect-batch`) in `detection-service/main.py` to offload synchronous, computationally blocking model inference functions (`run_sahi_inference`, `run_yolo_inference`, `run_depth_estimation`) to a background thread pool using `asyncio.to_thread()`. Also added memory of this change to `.jules/bolt.md`.

🎯 **Why:**
FastAPI routes defined with `async def` run on the main asyncio event loop. Calling blocking ML inference code directly blocks the entire event loop, preventing FastAPI from processing any other concurrent requests until the inference completes. This severely limits throughput. By wrapping the synchronous calls in `await asyncio.to_thread()`, we run them in a separate thread pool without blocking the main event loop, significantly increasing concurrency and throughput for concurrent requests.

📊 **Measured Improvement:**
A benchmark simulating 5 concurrent requests (where each simulated model inference blocked for 0.5s) was run:
- **Baseline:** The event loop processed requests sequentially because of the blocking call, yielding a total time of ~**2.519 seconds** for 5 requests.
- **Improved:** With `asyncio.to_thread()`, requests are processed concurrently, and the total response time for the same 5 concurrent requests dropped to ~**0.523 seconds**.
- **Change over baseline:** ~**79% reduction in total latency** for concurrent requests.
