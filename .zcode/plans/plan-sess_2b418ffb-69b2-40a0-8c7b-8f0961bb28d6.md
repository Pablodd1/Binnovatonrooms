## Tier 3: Mobile-First + Camera & PWA

### Problem Summary
The app works on desktop but has several broken features and poor UX on phones — the primary field-use device for construction inspection.

### 1. Touch Event Support (broken features)

**Compare slider** (`page.tsx:1034`) — uses only `onMouseMove`, completely non-functional on touch devices. Add `onTouchMove` and `onTouchStart` handlers. Enlarge the drag handle from 28px to 44px touch target.

**Heatmap tooltip** (`DefectHeatmap.tsx:140`) — uses only `onMouseMove`/`onMouseLeave`, tooltip never appears on phones. Add `onTouchStart` to show tooltip on tap.

### 2. Mobile Camera Toolbar Redesign

Currently the camera toolbar stacks 4 items vertically on mobile (~170px tall before the viewer even shows). Redesign for mobile:
- **Compact toolbar**: Camera toggle + Capture button as a fixed bottom bar overlay on the video viewer (like Instagram/Instagram camera UI)
- **Rear/Front camera toggle button** — currently hardcoded to `facingMode: "environment"`. Add a flip button that switches between `"environment"` and `"user"` constraints
- **Flash/Torch toggle** — use `MediaStream.applyConstraints({ advanced: [{ torch: true }] })` for dark construction sites (works on Android/Chrome, graceful fail on unsupported)
- Move "Comparar" and upload into a secondary actions sheet/drawer

### 3. Upload & Capture Row Optimization

Currently 5 buttons stack vertically on mobile (~250px). Restructure:
- **Upload button gets `capture="environment"`** on the file input — this enables direct iOS camera access from the "Subir imagen" button instead of opening the file picker
- **Primary action row**: "Analizar" button stays prominent, other actions (Export, PDF, Clear) collapse into a "..." overflow menu on mobile
- **Capture thumbnails**: Change mobile grid from 1-column to 3-column (`grid-template-columns: repeat(3, 1fr)` at 700px breakpoint) — cuts vertical space from 552px to ~184px

### 4. Touch Target Compliance

Increase all interactive element `min-height` from 42px to 46px for better mobile tap accuracy. This applies to all buttons, selects, and clickable elements.

### 5. PWA — "Add to Home Screen"

No manifest or service worker exists. Add:
- **`public/manifest.json`** — app name, short_name, theme_color, background_color, display: "standalone", icons (192x192 + 512x512 — generate placeholder SVGs)
- **`src/app/layout.tsx`** — add `<link rel="manifest">` via Next.js metadata, plus Apple PWA meta tags (apple-mobile-web-app-capable, apple-mobile-web-app-status-bar-style)
- **`public/sw.js`** — minimal service worker with cache-first for static assets, network-first for API calls. Enables offline loading of the UI shell.
- **Install prompt** — detect `beforeinstallprompt` event, show a subtle "Install app" banner

### 6. LiDAR & Depth — Honest UI Improvement

Real LiDAR (WebXR/ARKit depth) is **not available in any browser** — only native iOS apps can access LiDAR hardware. The current "LiDAR notes" textarea is a placeholder. Improve honestly:
- Rename "Notas LiDAR" to "Mediciones y profundidad" with better placeholder guidance
- Show the **Depth Anything V2 results from the Python service** prominently in the diagnosis results (surface depth stats: min/max/mean) — this data already comes back from `/detect` but is buried
- Add a "Thermal upload" hint — suggest users can upload FLIR thermal images via the upload button (accept already includes `image/*`)
- Note in the hardware section: "Para mediciones LiDAR nativas se requiere la app iOS nativa (en desarrollo)"

### 7. Report Detail Page Mobile Fix

- Add `flex-wrap: wrap` to `.report-detail-meta` to prevent horizontal overflow on narrow screens with long defect type names
