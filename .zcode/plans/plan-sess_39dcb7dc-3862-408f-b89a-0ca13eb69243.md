
# Admin Panel Full Control — Implementation Plan

## Problem Summary
The admin panel has no way to:
1. Create dedicated admin users (with username/password)
2. Grant/revoke admin access from the UI
3. Reset ANY user's password (currently just generates a link — doesn't email it)
4. Track all users with full management control

Also: **two disconnected admin systems** — the client (`settings/admins` emails) and server (`admins/{uid}` docs + `admin:true` claims) are never synced. This needs unification.

## Scope

### Part A: Unify the Admin System
The biggest bug: client-side admin check (email list) and server-side admin check (UID/claims/Firestore `admins` collection) are separate. A user added to `settings/admins` can SEE the admin panel but their API calls get rejected.

**Fix:** Make `settings/admins` the single source of truth. Update `verifyAdmin` to also check the email against `settings/admins`.

### Part B: Create Admin Users (New Feature)
Add a **"Create Admin Account"** capability:
- New API endpoint: `POST /api/admin/users` with `action: 'create_admin'`
- Creates a Firebase Auth user with email + temporary password
- Sets `admin: true` custom claim
- Adds email to `settings/admins` Firestore doc
- Auto-verifies the profile (bypasses vetting)
- Audit logs the creation

### Part C: Admin Management Panel (New UI)
New section in the Security tab — **"Admin Access Control"**:
- List all current admin users (from `settings/admins`)
- "Create Admin" button → form with email + temp password + name
- "Remove Admin" button per admin → removes from `settings/admins`, clears claim
- Shows when each admin was added and by whom

### Part D: Enhanced User Management
Upgrade the existing User Security panel:
- **"Reset Password" now works end-to-end** — generates reset link AND displays it with a "Copy Link" button + option to send via EmailJS
- **"Grant Admin"** button on any user → adds them to `settings/admins` + sets claim
- **"Change Password"** — admin sets a new password directly (not just a reset link)
- Fix missing audit log for email changes

### Part E: Remove Apple Sign-In
Since we're not paying for Apple Developer ($99/yr), remove the Apple button from Login.tsx to avoid confusing users with a broken button.

### Part F: Google Sign-In Setup Guide
No code change — just provide clear step-by-step documentation for the OAuth redirect URI fix (already provided verbally, will add as a setup doc).
